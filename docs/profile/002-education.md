# Education

Contains information about my education in the form of an autobiographical narrative. Do not treat this file as a final part of resume.

## Autobiographical Narrative

I studied in Ivanovo, Russia, at Ivanovo State Power Engineering University from September 2009 to July 2014. I completed a five-year Specialist degree program. I studied at the Faculty of Computer Science and Computer Engineering and at the Department of Computer Systems Software. The specialty or qualification listed in my diploma was "инженер-программист," which can be translated into English as "Software Engineer."

My graduation project was a real, working web portal rather than a purely academic prototype. By that time, I was already working at a company; as far as I remember, I joined the company during my third year of university. The project was a portal for conducting auctions of property owned by bankrupt companies. It was built as a monolithic web application using ASP.NET MVC 3 and MS SQL Server and used IIS as the web server. The application used Entity Framework for database access. The exact .NET Framework version is uncertain; I remember it as possibly .NET Framework 3.5, and I am certain that it was not .NET Framework 4. The frontend used JavaScript, jQuery, HTML from before HTML5, and CSS from before CSS3. I wrote the code in Visual Studio, and the project used SVN for version control, with simple commits to a shared repository. I believe I configured IIS myself, but this was long ago and I no longer remember the details well enough to document them. At that time, the system did not use Redis or message brokers.

As part of the transition from university into professional work, I completed an internship at the company where I later started working. After the internship, I joined the company during my third year of university. Details about the company and the role should be documented separately as work experience, but for the education narrative this internship is important because it connected my studies with my first professional software development work.

On the ASP.NET MVC 3 side, I implemented the whole application layer myself rather than only a specific component.

My work with MS SQL Server went beyond drawing the database diagram. I wrote SQL queries and, in order to improve application performance, moved a large part of the business logic into the database layer through stored procedures and functions. The business logic was deeply embedded in the database layer: even PDF document generation was handled through MS SQL Server using a special extension. I no longer remember the name of that MS SQL Server extension.

I also handled MS SQL Server deployment and maintenance from scratch. This included server setup, configuration, access rights, backups, and scheduled backup jobs using Windows Task Scheduler. As a result, my database work included both development and operational support.

The project included integration with Russian CryptoPro cryptographic systems in order to meet legal requirements related to electronic digital signatures. I remember using a CryptoPro library for C# and a JavaScript library on the frontend. All documents were signed through this system, but I do not remember the implementation details well enough to document them precisely. The portal went live, passed the required state audits, and was used in real work. As far as I understand, the portal is still available publicly, although it has probably been rewritten many times since my original work.

My role covered both software engineering and analysis activities. I gathered requirements, prepared use cases, created UML diagrams, wrote technical documentation, designed the database diagram, and created the deployment diagram. The diagrams included class diagrams, use case diagrams, a database diagram, a deployment diagram, and flowcharts. I used PlantUML for diagrams and related documentation work; at the time I used it in 2014, I remember it as a very new tool. The project was supervised by a more senior colleague at work, who reviewed my work, but I was the only person implementing the code. Since I was the only developer on this project, the workflow was informal: I wrote the code as needed and received review and guidance from the senior colleague. This reflected the 2013-2014 context I worked in: in my environment in Russia at that time, modern development-process vocabulary and practices such as Agile, Scrum, Kanban, Git-based workflows, and formalized team workflows were not yet part of the everyday working culture. Testing was manual; I did not write automated tests at that time. In that sense, it was a developer project carried out under senior supervision.

The thesis defense had an unusual practical constraint. Under Russian legal requirements, I could not take the flash drive containing the electronic digital signature outside the office where the development took place. Because access to the portal was possible only with an electronic digital signature, I showed the diploma committee the diagrams, part of the code, and only the login page of the working system. I successfully defended the diploma project.

One of the more interesting course or laboratory projects I completed at university was written in MASM assembly language. The project implemented calculations with ultra-high-precision floating-point numbers, with each number taking approximately 2 KB of memory. The byte-level representation of the numbers was provided by the instructor, while I designed the algorithms myself. The module implemented basic mathematical operations: addition, subtraction, division, multiplication, square root, and exponentiation. I packaged the module as an x86 DLL, worked with DLL exports, and created a library that could be embedded into C++ and C# projects. I had the impression that the project may have been related to the defense industry, but I am not certain about that, so I treat it only as a personal assumption.

Another course project I remember well was for a programming language theory course. We had a Prolog course where the original expectation was to complete laboratory assignments in an old Prolog environment, which I remember as a DOS application. The proposed browser-based setup relied on an Internet Explorer ActiveX extension, but it did not work in practice. For my course project, I wrote a Prolog interpreter in JavaScript that could run Prolog code directly on a browser page. As far as I remember, it fully supported the basic part of Prolog that we used in the course: all Prolog code we wrote for the laboratory assignments could run in my browser-based interpreter. I do not remember whether it supported optional extensions or any non-basic parts of the language.

The curriculum had a strong focus on mathematics and foundational computer science. The relevant subjects I remember include higher mathematics, computational mathematics, function optimization theory, combinatorics, fuzzy logic, data structures and algorithms, databases, programming language theory, Prolog, assembly language, C# programming, software engineering, and UML.

There was also a dedicated course on website development. In that course, I wrote websites using PHP 4 and MySQL. The coursework included sessions, forms, and authentication. Authentication used HTTP-only cookies rather than JWT. I also administered the Apache server and MySQL in that context.

There was also a dedicated course on computer network topology. In that course, we deployed a local network in a classroom, laid cables and fiber optic cable ourselves, and configured the network. As far as I remember, administration was done through Linux servers running Arch Linux. The Linux and networking work included shell scripting and DNS, but not firewall work.

The program also included a separate Linux course, where we studied Arch Linux and Linux architecture, as well as a course on processor architecture. Outside the core IT subjects, I also studied physics, strength of materials, and philosophy.

Overall, my education gave me practical skills in algorithmic thinking, low-level programming, database design and development, backend development, systems administration, network configuration, technical documentation, UML modeling, requirements analysis, and manual testing.

The connection between this education and my current career direction is not limited to specific programming languages or technologies. The most valuable outcome was that the university taught me a systematic approach to software development. It gave me a strong theoretical foundation that repeatedly helped me write good, maintainable, working code. Many practices I still use in professional work, including code design, application design, and general engineering principles such as SOLID and DRY, grew out of that university foundation.

This education also changed the way I think more broadly. It helped me move from everyday intuitive thinking toward systematic thinking: decomposing problems, analyzing constraints, and looking for workable solutions. That way of thinking has been useful not only in programming, but also in real life. My first professional role after the internship was a junior fullstack developer role, and over roughly six years at that company I grew to a senior level; the details of that growth belong in the work experience section, but the foundation for it started during my education.

For international context, the Russian Specialist degree should be described carefully. It is not literally the same diploma as a master's degree, but Russian education law places specialist programs and master's programs at the same level of higher education. A concise English explanation is: "Specialist degree, a five-year Russian higher education credential at the same national higher-education level as a master's degree."

## Publicly Verified Context

- Federal Law No. 273-FZ "On Education in the Russian Federation," Article 10, lists "higher education - specialist degree, master's degree" as one level of professional education.
- Federal Law No. 273-FZ, Article 60, states that higher education at the specialist level is confirmed by a specialist diploma and higher education at the master's level is confirmed by a master's diploma.
- Publicly verified sources used: ConsultantPlus pages for Article 10 and Article 60 of Federal Law No. 273-FZ.
- Public release history sources indicate that .NET Framework 3.5 existed well before the 2013-2014 graduation project period; the exact .NET Framework version used in the project remains unconfirmed, but I am certain that it was not .NET Framework 4.
